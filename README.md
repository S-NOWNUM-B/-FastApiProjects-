# Репозиторий проектов на FastAPI

Добро пожаловать в мой репозиторий, посвящённый проектам на **FastAPI**! Здесь собрана коллекция образовательных и практических примеров, иллюстрирующих разработку REST API с использованием **FastAPI**, **PostgreSQL** и других современных технологий.

## Содержание проектов

В репозитории представлены следующие проекты:

1. **FastApiProject_1-3**: Базовые проекты с простой структурой и основными функциями FastAPI
2. **FastApiProject_4-6**: Проекты с добавленной аутентификацией и авторизацией
3. **FastApiProject_7-9**: Проекты с роутерами и более сложной структурой
4. **FastApiProject_10-12**: Проекты с Docker и Docker Compose
5. **FastApiProject_13-15**: Проекты с WebSocket и Redis
6. **FastApiProject_16-18**: Проекты с Alembic для миграций базы данных
7. **FastApiProject_19-20**: Полноценные проекты с логированием, мониторингом и CI/CD

## Технологии

В проектах используются следующие технологии:

- **FastAPI**: Современный веб-фреймворк для создания API
- **PostgreSQL**: Реляционная база данных
- **SQLAlchemy**: ORM для работы с базой данных
- **Pydantic**: Валидация данных и сериализация
- **Alembic**: Управление миграциями базы данных
- **Redis**: Кэширование и работа с WebSocket
- **Docker**: Контейнеризация приложений
- **Prometheus**: Мониторинг приложения
- **JWT**: Аутентификация и авторизация
- **WebSocket**: Реал-тайм коммуникация
- **Pytest**: Тестирование API

## Инструкции по запуску

Чтобы начать работу с проектами, выполните следующие шаги:

1. **Клонирование репозитория**:  
   Склонируйте репозиторий на локальную машину и перейдите в папку нужного проекта:
   ```bash
   git clone <адрес_репозитория>
   cd <папка_проекта>
   ```

2. **Создание виртуального окружения**:  
   Настройте виртуальное окружение и установите зависимости:
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # Для Linux/macOS
   .venv\Scripts\activate     # Для Windows
   pip install -r requirements.txt
   ```

3. **Конфигурация базы данных**:  
   Отредактируйте файлы проекта, указав параметры подключения к **PostgreSQL**:
   ```python
   DATABASE_URL = "postgresql+asyncpg://username:password@localhost:5432/database_name"
   ```

4. **Запуск с Docker** (если поддерживается):
   ```bash
   docker-compose up --build
   ```

5. **Запуск без Docker**:
   ```bash
   uvicorn main:app --reload
   ```

6. **Применение миграций** (если есть):
   ```bash
   alembic upgrade head
   ```

7. **Тестирование API**:  
   - Откройте Swagger UI: `http://localhost:8000/docs`
   - Или ReDoc: `http://localhost:8000/redoc`
   - Используйте Postman или curl для тестирования

## Рекомендации и лучшие практики

### Безопасность
- Храните секреты в переменных окружения
- Используйте HTTPS в продакшене
- Валидируйте все входные данные
- Используйте rate limiting для API
- Регулярно обновляйте зависимости

### Производительность
- Используйте асинхронные операции
- Применяйте кэширование где возможно
- Оптимизируйте запросы к базе данных
- Используйте connection pooling
- Мониторьте производительность

### Код
- Следуйте PEP 8
- Используйте типизацию
- Пишите тесты
- Документируйте код
- Используйте линтеры и форматтеры

### DevOps
- Настройте CI/CD
- Используйте контейнеризацию
- Настройте мониторинг
- Ведите логи
- Автоматизируйте деплой
